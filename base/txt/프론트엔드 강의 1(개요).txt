html (Hyper Text Markup Language) - 기획자 / 웹의 구조를 담당 - 구조
css (Cascading Style Sheets) - 디자이너 / 시각적인 표현(정적)을 담당 - 스타일
js (JavaScript) 개발자 / 동적 처리를 담당 - 동적인 움직임

웹앱의 동작원리
url -> 서버로 요청 -> 브라우저 사용자의 모니터에 응답 -> 추가요청 -> 추가 응답(js, css, img ....)

웹 표준과 브라우저
w3c  - 웹표준
브라우저 공급업체 웹표준을 해석하고 만듬
크로스 브라우징 - 크롬, 엣지, 파이오폭스, 오페라. 웨일, 사파리 등등 구동되는 여러 브라우저에서 동일한 사용자 경험을 줄 수 있도록 제작하는 기술, 방법

viewport - 웹페이지가 출력되는 영역

웹에서 사용하는 이미지
비트맵 - 픽셀이 모여 만들어진 정보의 집합, 레스터 이미지라고도 부름 / jpg, png 사진, 그림
  1)정교하고 다양한 색상을 자연스럽게 표현
  2) 확대.축소 시 계단 현상, 품질 저하

벡터 - 점, 선, 면의 위치 색상등 수학적 정보의 형태 / svg, icon 로고. 아이콘
  1) 정교한 이미지를 표현하기 어려움
  2) 확대. 축소에서 자유로움, 용량 변화가 없음

jpg - 손실 압축 
png - 비손실 압축 ,alpha channel 지원(투명 배경)
gif -움짤, 애니메이션
webp - 완벽한 손실/비손실 압축 지원, gif 같은 애니메이션 지원
alpha channel 지원 (손실, 비손실 모두) 최근에 나와서 지원되는 브라우저가 따로 있음
svg - 마크업 언어 기반의 벡터 그래픽을 표현, 해상도의 영향에서 자유로움, css와 js로 제어 가능, 파일 및 코드 삽입 가능

특수문자용어
Backtick, Grave 백틱, 그레이브 -> ` (1번 왼쪽)
Tilde 틸드, 물결표시 -> ~
Exclamation mark 엑스클러메이션, 느낌표 -> !
At sign 앳, 콜뱅이 -> @
Sharp, Number sign 샵, 넘버, 우물 정 -> #
Dolloar sign 달러사인 -> $
Percent sign, 퍼센트 -> %
Caret, 캐럿 -> ^ (~ 이상)
Ampersand, 엠퍼센드 -> &
Asterisk , 별 ->? *
Hyphen, Dash 하이픈, 대시, 마이너스 ->  -
Underscore, Low dash ,언더스코어, 로대시, 밑줄 -> _
Equals sing, 이퀄, 동등 -> =
Quotation mark , 퀘테이션, 큰 따옴표 -> "
Apostroghe 아포스트로피, 작은 따옴표 -> '
Colon ,콜론 -> :
Semicolon , 세미콜론 -> ;
Comma, 콤마 -> ,
Period, Dot , 피리어드, 닷, 점, 마침표 -> .
Qusetion mark , 퀘스천, 물음표 -> ?
Slash, 슬래시 -> /
Vertical bar , 버티컬 바-> |
Backslash, 백슬래시, 역 슬래시 -> \
Parenthesis, 퍼렌서시스, 소괄호, 괄호 -> ()
Brace, 브레이스, 중괄호 -? {}
Bracket, 대괄호 -? []
Angle Bracket, 앵글 브래킷, 꺽쇠괄호 -> <>
